Baseball305

As sports fans, we thought it would be fun to design a Baseball simulator game. As computer science students the idea of a baseball simulator game is almost endlessly expandable. The spectrum of complexity is vast for this project, and continuosly improving the project is our goal.

The Baseball Simulator game has all 30 MLB teams. The user can choose 2 teams and begin the simulation. Using individual players' skill rating, and a realistic implementation of randomness, each at bat is simulated. The game will print out status updates similar to a TV graphic you would see on a baseball game. The game will also print out the final box score at the end of the game showing when the runs were scored throughout the game.

The project is designed such that there is a overall simulator, GameSimulator that controls the flow of exectuion of the simulation. The SetupGame class handles all user input to select the teams, and loads the team information in from the resources file, and creates a game object. We decided to make a non-static Game Class, for future expansion, in order to simulate more than one Game. Each Team consists of a starting pitcher, and a batting lineup. Each team has 1 Pitcher and 9 Batters, and they each have skill ratings associated, found from MLB The Show 2024. For each at-bat, the program calculates the skill difference rating of the pitcher and batter and determines a hit probability. A random number is generated, and using the hit probability, is either a hit or an out. If it is a hit, then it is determined what type of hit it is (40% Single, 30% Double, 20% Home Run, 10% Triple). The user is prompted to either simulate the next at bat only, the rest of the inning, or the entire game. The result of each atbat is printed to the screen along with the current game status at each new at bat. When the game is over a box score is printed indicating the runs scored by each team, per inning.

Implementation: This project provided a great use case for test driven development, and specifically the designing of the project before coding. My partner and I spent a significant amount of time planning out the design of the project before writing any code, which was a new process for me.
